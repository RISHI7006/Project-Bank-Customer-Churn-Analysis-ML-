# Bank Churn Risk Intelligence — Streamlit App

A production-ready Streamlit app for the churn model built in
`Bank_Customer_Churn_Analysis.ipynb`. Includes single-customer scoring,
batch CSV scoring, model comparison, and SHAP-based explainability.

## Files

```
churn_app/
├── app.py               # Streamlit app (the deployed entry point)
├── train.py             # Rebuilds models from the raw CSV, saves models/
├── requirements.txt     # Pinned dependencies for Streamlit Cloud
├── .gitignore
├── models/               <- generated by train.py, commit this to the repo
│   ├── best_model.pkl
│   ├── xgboost_model.pkl
│   ├── logreg_model.pkl
│   ├── scaler.pkl
│   ├── feature_columns.pkl
│   ├── numeric_features.pkl
│   ├── meta.json
│   └── train_reference.csv
└── European_Bank.csv    # raw dataset (only needed to run train.py)
```

## 1. Run it locally first

```bash
# 1. Create an isolated environment
python -m venv .venv
source .venv/bin/activate          # Windows: .venv\Scripts\activate

# 2. Install dependencies
pip install -r requirements.txt

# 3. Put your raw dataset next to train.py, then train + save artifacts
python train.py --data European_Bank.csv

# 4. Launch the app
streamlit run app.py
```

Open the URL Streamlit prints (usually `http://localhost:8501`) and confirm
all four pages (Single Customer, Batch Scoring, Model Performance, Key
Insights) work before you deploy.

> **Important:** `train.py` needs `European_Bank.csv` (the dataset used in
> the notebook). `app.py` does **not** need the CSV — it only needs the
> `models/` folder that `train.py` produces. Train once locally, commit the
> resulting `models/` folder, and Streamlit Cloud just serves the app.

## 2. Push the project to GitHub

Streamlit Community Cloud deploys directly from a GitHub repo.

```bash
cd churn_app
git init
git add app.py train.py requirements.txt .gitignore models/
git commit -m "Bank churn risk app"
git branch -M main
git remote add origin https://github.com/<your-username>/bank-churn-app.git
git push -u origin main
```

Notes:
- Commit the `models/` folder — Streamlit Cloud does **not** run `train.py`
  for you, and it has no access to your local `European_Bank.csv`.
- Individual `.pkl` files here are small (well under GitHub's 100 MB limit).
  If your real dataset/model grows large, use [Git LFS](https://git-lfs.com/).
- Do **not** commit secrets (API keys, credentials) — use Streamlit's secrets
  manager instead (see step 4).

---

## 3. Deploy on Streamlit Community Cloud

1. Go to **https://share.streamlit.io** and sign in with GitHub.
2. Click **"Create app"** → **"Deploy a public app from GitHub"**.
3. Fill in:
   - **Repository:** `<your-username>/bank-churn-app`
   - **Branch:** `main`
   - **Main file path:** `app.py`
4. (Optional) Click **"Advanced settings"** to:
   - Pin the **Python version** (e.g. 3.11) to match what you tested locally.
   - Add any **Secrets** in TOML format if you later add API keys, DB
     credentials, etc. (not required for this app as-is).
5. Click **"Deploy"**. Streamlit Cloud will:
   - Clone your repo.
   - Create a fresh environment and `pip install -r requirements.txt`.
   - Run `streamlit run app.py`.
6. Build logs stream live in the browser. First deploy typically takes
   2–5 minutes (installing `xgboost`/`shap` takes the longest).
7. Once it says **"Your app is live!"**, you'll get a public URL like:
   `https://bank-churn-app-<hash>.streamlit.app`

---

## 4. Update the app later

Any `git push` to the connected branch triggers an automatic redeploy:

```bash
git add .
git commit -m "Update model / UI"
git push
```

To retrain on new data: rerun `python train.py --data new_data.csv` locally,
commit the refreshed `models/` folder, and push — no code changes needed.

---

## 5. Common deployment issues

| Symptom | Fix |
|---|---|
| `ModuleNotFoundError: No module named 'xgboost'` | Confirm `requirements.txt` was committed at the repo root (or same folder as `app.py`) and matches the versions above. |
| App shows "Missing model artifacts" | You forgot to commit the `models/` folder, or `train.py` was never run. Re-run `train.py` and `git add models/ -f` if `.gitignore` is excluding it. |
| Build works but times out on `shap` install | `shap` compiles native extensions; the pinned version above (`0.46.0`) has prebuilt wheels for Streamlit Cloud's Linux/Python combo — avoid unpinning it. |
| App is slow on first load | Normal — model files + SHAP explainer load once and are cached via `@st.cache_resource`. Subsequent interactions are fast. |
| "File size too large" on GitHub push | Use Git LFS for any artifact over ~50 MB, or retrain with fewer estimators to shrink `.pkl` size. |

## App pages

- **Single Customer** — interactive sliders/inputs, live churn-probability
  gauge, and a SHAP bar chart showing which features pushed the score up or
  down for that specific customer.
- **Batch Scoring** — upload a CSV of many customers, get back churn
  probabilities + risk tiers, and download the scored file.
- **Model Performance** — accuracy/precision/recall/F1/ROC-AUC table, grouped
  bar chart, ROC curves, and feature importance for all five trained models.
- **Key Insights** — the business findings and recommendations from the
  notebook's EDA section.
